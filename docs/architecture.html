Filter Data Flow

01
User Selects Filter
FilterBar.vue updates one of 4 reactive refs in useFilters.js
02
View Reacts
Each view watches filter state and calls getCurrentFilters() to build query object
03
API Call
api.js builds URLSearchParams, skipping any value equal to 'all'
04
Backend Filters
apply_filters() and filter_by_month() narrow the in-memory arrays. Quarters expand to month lists via QUARTER_MAP
05
Computed Re-renders
Raw data stored in ref(); all derived values in computed() update automatically
The 4-Filter System
warehouse category status month

All four filters propagate as query params. The backend treats 'all' as "no filter". Month accepts direct values (2025-03) or quarter aliases (Q1-2025). Inventory endpoints do not support month — inventory has no time dimension. Spending and demand endpoints accept no filters at all.

Frontend State (Composables)

useFilters.js
Singleton. Holds the 4 active filter values. All views share one instance — changing a filter anywhere updates everywhere.
  • selectedPeriod
  • selectedLocation
  • selectedCategory
  • selectedStatus
  • getCurrentFilters()
  • resetFilters()
  • hasActiveFilters
useAuth.js
Singleton. Mock user session with language-aware name, title, department, and task list.
  • currentUser (computed)
  • logout()
  • getInitials(name)
useI18n.js
Singleton. EN/JA translation with localStorage persistence. Currency auto-selects USD or JPY.
  • t(key, params)
  • setLocale(locale)
  • currentLocale
  • currentCurrency
  • translateProductName()
  • translateWarehouse()
